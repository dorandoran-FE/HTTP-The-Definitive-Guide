## 6장 프락시

### 6.1 웹 중개자

- 게이트웨이 (HTTP -> FTP)
- 프록시 (HTTP -> HTTP)

### 6.2 왜 프락시를 사용하는가?

- 보안
- 추가 기능
- 유해컨텐츠 차단
- 성능 (캐시)

### 6.3 프락시는 어디에 있는가?

- 클라이언트 출구
- 서버 앞
- 인터샙트

### 6.4 클라이언트 프락시 설정

- 수동
- 자동

### 6.5 프락시 요청의 미묘한 특징들

- URI (웹서버 축약 버전 vs 프록시 full url)
- 인터샙터 uri (클라이언트의 요청 가로챔)
